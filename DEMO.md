# 多媒体展示平台 - Web 后台管理系统前端设计文档

## 1. 项目概述
* **项目名称**：多媒体展示小软件（类似朋友圈）后台管理系统
* **负责模块**：Web 端管理后台（前端开发）
* **技术栈建议**：React + TypeScript + Ant Design + ECharts
* **核心目标**：为管理员提供对用户、内容（图片/视频/文本）的全面监管能力，并展示系统运行数据的可视化报表。

---

## 2. 页面路由结构 (Sitemap)

| 序号 | 页面名称 | 路由路径 (Route) | 功能简述 |
| :--- | :--- | :--- | :--- |
| 1 | **登录页** | `/login` | 管理员身份认证 |
| 2 | **数据仪表盘** | `/dashboard` | **(加分项)** 系统核心数据统计与可视化 |
| 3 | **用户管理** | `/users` | 用户查询、封禁、状态管理 |
| 4 | **动态内容管理** | `/posts` | **(核心)** 图文/视频内容的审核与管理 |
| 5 | **敏感词设置** | `/settings/sensitive` | **(加分项)** 违规词库配置 |

---

## 3. 详细功能模块设计

### 3.1 登录页 (Login)
* **布局**：页面居中卡片式布局，背景建议使用科技感或简洁风格图片。
* **核心组件**：
    * `UsernameInput`: 管理员账号输入框。
    * `PasswordInput`: 密码输入框（加密传输）。
    * `LoginButton`: 登录按钮，点击触发 API 请求。
* **逻辑处理**：
    * 前端表单校验（非空判断）。
    * 登录成功后，将 Token 存入本地 Storage，并重定向至 Dashboard。

### 3.2 数据仪表盘 (Dashboard) —— [加分项：统计分析]
此页面用于展示系统整体活跃度和健康状况，体现“大模型处理”或“活跃度”统计结果。

* **顶部数据卡片 (Data Cards)**：
    1.  **用户总数**：展示当前注册用户总量（及较昨日新增数）。
    2.  **动态总数**：平台累计发布的帖子数量。
    3.  **待审核内容**：被 AI 模型标记为“疑似违规”的数量。
    4.  **今日访问量**：当日 API 调用次数或 PV。

* **可视化图表 (Charts)**：
    * **用户增长趋势图** (折线图)：展示过去 7 天/30 天的新增用户变化。
    * **内容类型占比** (饼图)：展示 图片贴 vs 视频贴 vs 纯文本贴 的比例。
    * **热门标签词云** (词云图/排行)：展示小程序端最常用的标签 Top 10。

### 3.3 用户管理 (User Management)
对小程序端注册用户进行监管。

* **顶部筛选栏**：
    * 搜索框：支持按“昵称”或“手机号”模糊搜索。
    * 状态筛选：全部 / 正常 / 已封禁。
    
* **数据表格 (Table Columns)**：
    * **头像**：显示圆形缩略图 (`Avatar` 组件)。
    * **昵称/ID**：用户基本标识。
    * **注册时间**：格式化为 `YYYY-MM-DD HH:mm`。
    * **发布动态数**：统计该用户发帖数量。
    * **当前状态**：使用不同颜色的 Tag 展示（如：绿色-正常，红色-封禁）。
    * **操作列**：
        * `详情`：点击弹出抽屉，查看用户详细画像。
        * `封禁/解封`：点击弹出确认框，输入封禁理由后执行操作。
        * `重置密码`：(可选) 辅助用户找回密码。

### 3.4 动态内容管理 (Post Management) —— [核心 & 技术难点]
对用户发布的多媒体内容进行审核与管理。

* **顶部筛选栏**：
    * 发布时间范围筛选。
    * 内容标签筛选（如：生活、学习、娱乐）。
    * **审核状态筛选**：全部 / AI 疑似违规 / 正常。

* **数据表格 (Table Columns)**：
    * **发布者**：显示发布用户昵称。
    * **内容摘要**：展示文本内容的前 30 个字，超长省略。
    * **媒体预览 (重点功能)**：
        * **图片**：表格中显示 50px 小图，点击触发 `Image.Preview` 全屏查看大图。
        * **视频**：表格中显示视频封面图 + 播放图标。点击弹出 `Modal` 模态框，在框内播放视频流。
    * **标签 (Tags)**：展示用户给帖子打的标签。
    * **AI 安全评分 (加分项)**：后端集成大模型或敏感词过滤后的结果（如：显示“高风险”红色标签）。
    * **发布时间**：`YYYY-MM-DD HH:mm`。
    * **操作列**：
        * `下架/删除`：**强制要求**管理员选择或填写下架理由（如：涉及敏感信息、垃圾广告），点击确认后软删除该条数据。
        * `置顶`：(可选) 将优质内容在小程序端置顶显示。

### 3.5 敏感词设置 (Sensitive Words) —— [加分项：过滤功能]
配合后端的敏感词过滤逻辑，提供可视化的管理界面。

* **功能区域**：
    * **添加词汇**：输入框 + 添加按钮。输入如“暴力”、“赌博”等词汇，回车通过。
    * **词库列表**：使用标签组 (`Tag Group`) 展示所有已添加的敏感词。
    * **删除词汇**：每个标签带有关闭图标 `x`，点击即可从词库移除。
    * *注：此处数据的变动应实时同步给后端，影响小程序端的发帖检测逻辑。*

## 4. 非功能性要求 (UI/UX)
* **界面风格**：整体界面需保持工整，推荐使用 Ant Design Pro 或 Arco Design 的标准布局。
* **响应体验**：操作（如删除、封禁）需有二次确认 (`Modal.confirm`)，操作成功需有全局提示 (`Message.success`)。
* **加载状态**：表格数据加载、图表渲染时需展示 Loading 骨架屏。
